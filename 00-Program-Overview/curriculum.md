# 📋 GRC Mentorship Programme — Full Curriculum Overview

## Programme Philosophy

This programme is built on one principle: **you learn GRC by doing GRC**.

Every week you produce professional deliverables — risk registers, gap assessments, audit reports, board presentations — exactly as a GRC Analyst or Consultant would in a real organization. By the end of 24 weeks, you have a portfolio that proves your competence.

---

## How Each Week Works

```
Monday       → Read week README.md and assignment.md
Tuesday-Wed  → Complete the scenario research and planning
Thursday-Fri → Produce the professional deliverables
Saturday     → Mentor session (90 min): review + feedback
Sunday       → Upload to GitHub and prepare for next week
```

---

## Mentorship Sessions

Each week includes **2 mentorship sessions**:

| Session | Format | Duration | Focus |
|---------|--------|----------|-------|
| Session A | Live call / recording | 90 min | Topic teaching + exercise |
| Session B | Live review call | 60 min | Assignment feedback |

**Mentor responsibilities:**
- Provide written feedback on every assignment
- Share real-world examples and case studies
- Conduct mock interviews (Weeks 4, 8, 12, 16, 20, 24)
- Review GitHub portfolio quality
- Provide certification study guidance

## Grading Scale

| Grade | Score | Description |
|-------|-------|-------------|
| 🏆 Distinction | 90–100% | Board-ready quality. No feedback required. |
| ✅ Pass | 70–89% | Professional quality. Minor improvements needed. |
| 🔄 Resubmit | 50–69% | Significant gaps. Resubmit within 1 week. |
| ❌ Incomplete | <50% | Assignment not attempted or fundamentally incomplete. |

## The GitHub Portfolio Standard

Your GitHub portfolio must meet these standards to be considered job-ready:

### Every Repository Must Have:
- ✅ Professional README.md with scenario, frameworks, and skills demonstrated
- ✅ All deliverables in the correct folder structure
- ✅ No placeholder text or "Lorem ipsum" content
- ✅ Professional document formatting (version headers, company logos, review dates)
- ✅ Framework references clearly cited in all documents

### Portfolio Quality Checklist (Week 24 Final Review):
- [ ] 15+ projects with complete deliverables
- [ ] Consistent professional formatting across all projects
- [ ] Capstone project is the flagship — most complete and polished
- [ ] README.md for each project clearly explains the scenario and your role
- [ ] All templates used are populated (not blank templates)
- [ ] GitHub profile has professional bio mentioning GRC specialisation
