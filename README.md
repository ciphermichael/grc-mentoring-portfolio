## About This Repository

This repository is the official learning hub for the **6-Month Cybersecurity GRC Mentorship Program** — a structured, portfolio-first programme that trains complete beginners to become job-ready GRC Analysts through 24 weeks of real-world simulation.

Every week produces professional-grade deliverables — risk registers, gap assessments, audit reports, policies, vendor assessments, and board presentations — exactly as a GRC Analyst would produce them in a real organisation. By programme end, you have a GitHub portfolio of 15 completed projects that proves your competence to any employer.

**What this repository contains:**

- **24 weekly learning guides** — concept explanations, real-world examples, framework deep dives, common mistakes, professional tips, and week-specific resources
- **24 fully detailed assignments** — complete scenario briefs, step-by-step deliverable instructions, quality bars, and example content
- **15 portfolio-grade GRC projects** across all phases
- **13 professional templates** ready for real-world use (risk registers, gap assessments, audit checklists, board reports, and more)
- **Sample completed evidence files** in Excel and PowerPoint format
- **A complete capstone project** — build an enterprise GRC programme for NovaTech Corp from scratch
- **Interview preparation** built into every week (10–12 questions + STAR-format answers)
- **Master resources guide** — 100+ curated learning resources, books, tools, communities, and job boards

## How Each Week Works

Each week folder contains three files that work together:

**`learning-guide.md`** — Read this first. Contains:
- Core concept explained with real-world analogies
- Step-by-step breakdown of how GRC professionals apply the concept
- Framework deep dive with specific clause/article/control references
- Common mistakes beginners make and how to avoid them
- Professional tips from experienced GRC practitioners
- 6–8 week-specific resources (different URLs every week)
- Instructor mentorship guidance (Session A and Session B agendas)
- 10–12 interview questions with STAR-format example answers

**`assignment.md`** — The work. Contains:
- Full scenario brief (company background, current problems, your role, stakeholders)
- Step-by-step instructions for every deliverable
- Quality bar defining what Distinction-level work looks like
- Example content with realistic sample data for the client
- Week-specific resources

**`submission-checklist.md`** — Before you submit. Quality checks and self-assessment.

## Getting Started

### Week 1 Checklist

- [ ] Fork this repository to your GitHub account
- [ ] Set up your GitHub profile (professional photo; bio mentioning GRC specialisation)
- [ ] Read `00-Program-Overview/curriculum.md` — full programme overview
- [ ] Read `00-Program-Overview/master-resources.md` — bookmark key resources
- [ ] Read `Week-01/learning-guide.md` — understand the concepts before starting
- [ ] Complete the `Week-01/assignment.md` deliverables
- [ ] Upload to GitHub and update your `Week-01/README.md` completion status

### How to Work Each Week

1. **Start with the learning guide** — read `learning-guide.md` fully before opening the assignment
2. **Read the assignment brief carefully** — understand the client scenario before producing any deliverables
3. **Check the templates folder** — most assignments reference a template; use it
4. **Produce board-ready work** — everything goes into your public portfolio; treat every deliverable as if a real client will read it
5. **Commit to GitHub regularly** — active commit history demonstrates professionalism
6. **Tag your mentor** for review using `@mentor-name` in your commit or README update
